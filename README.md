# Ecommerce Survey Form

## Project Overview

This project is a simple Ecommerce Product Survey Form created using only HTML.
The purpose of this project is to practice HTML structure, forms, tables, inputs, buttons, and semantic elements without using CSS or JavaScript.

---

## Features

* Product information section
* Quantity input field
* Add to Bag button
* Newsletter signup offer
* Customer survey form
* Organized layout using HTML tables
* Semantic HTML elements like:

  * `<fieldset>`
  * `<legend>`
  * `<form>`
  * `<table>`
  * `<input>`
  * `<button>`

## Code Structure Explanation

### 1. HTML Document Setup

The project starts with the basic HTML structure:

```html
<html>
<head>
<title>Ecommerce Product</title>
</head>
<body>
```

This defines the webpage and browser title.

---

### 2. Table Layout

A table is used to organize product details and form sections.

```html
<table>
<tr>
<td>
```

This keeps the content aligned in rows and columns.

---

### 3. Product Quantity Input

The quantity selector uses:

```html
<input type="number" value="1" min="1">
```

* `type="number"` allows numeric input
* `min="1"` prevents negative values

---

### 4. Buttons

Buttons are added for interaction:

```html
<button>Add to Bag</button>
<button>Sign Up Now</button>
```

These represent ecommerce actions.

---

### 5. Offer Section

The special offer uses semantic tags:

```html
<fieldset>
<legend>Limited-Time Offer!</legend>
```

This groups related content in a clean way.

---

## Design Decisions

### Why Only HTML?

This project focuses on learning:

* HTML forms
* Semantic structure
* Input handling
* Content organization

without relying on CSS or JavaScript.

---

### Why Use Tables?

Tables help create a simple aligned layout for beginners and make the content easier to structure.

---

### Why Use Fieldset and Legend?

These tags improve:

* readability
* accessibility
* semantic structure

especially for grouped form sections.
